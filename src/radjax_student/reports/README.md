# Reports

Reports make claims explicit.

This package is the long-term home for run reports, validation reports,
evaluation reports, diagnostics, and claims-not-made sections. Passing a smoke
test is not evidence of model quality.

Phase 1 provides `StudentInspectionReport` and `StudentDoctorReport`. They
aggregate existing artifact, defaults, and compatibility objects for human and
JSON rendering; they do not create competing validation policy or execution
behavior.
